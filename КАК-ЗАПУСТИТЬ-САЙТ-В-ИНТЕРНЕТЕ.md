# Как запустить сайт на хостинге и в интернете

## Что нужно для запуска

1. **Файлы сайта** — у вас уже есть: `index.html`, `css/`, `js/`, `img/`, `api/`.
2. **Домен** (по желанию) — чтобы сайт открывался по своему адресу (например `agressor.ru`). Без домена будет бесплатный поддомен хостинга (например `ваш-проект.vercel.app`).
3. **Работа форм** — для отправки заявок в Telegram нужны:
   - Telegram-бот (создать у [@BotFather](https://t.me/BotFather), получить токен).
   - ID чата (написать боту [@userinfobot](https://t.me/userinfobot) — он покажет ваш ID).
   - На хостинге указать переменные: `TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID`.

---

## Общий порядок действий

1. Выбрать хостинг из списка ниже.
2. Зарегистрироваться и создать проект/сайт.
3. Загрузить папку проекта (через Git, ZIP или перетаскивание).
4. Если хостинг поддерживает serverless (как Vercel/Netlify) — настроить переменные окружения для Telegram.
5. При желании подключить свой домен в настройках проекта.

---

## Лучшие варианты хостингов: цена и качество

Для вашего лендинга (статический сайт + одна API-функция для заявок) подойдут платформы с бесплатным тарифом и поддержкой serverless. Ниже — варианты, где сайт будет «конфеткой»: быстро, стабильно, дёшево или бесплатно.

---

### 1. **Vercel** (оптимально для вашего проекта)

- **Цена:** бесплатно (тайр Hobby); платно от ~$20/мес при росте.
- **Плюсы:**  
  - У вас уже есть `vercel.json` и `api/send-lead.js` — проект заточен под Vercel.  
  - Очень быстрая отдача страниц, CDN по миру.  
  - Бесплатный SSL, свой поддомен `ваш-проект.vercel.app`.  
  - Serverless-функции из папки `api/` работают «из коробки».  
- **Минусы:** для продакшена с кастомным доменом бесплатный тариф обычно достаточен; ограничения по трафику/функциям при росте.
- **Как запустить:** установить [Vercel CLI](https://vercel.com/docs/cli), в папке проекта выполнить `vercel`, в Dashboard добавить `TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID`.
- **Итог:** лучший вариант «под ключ» для вашего стека: сайт будет быстрым и аккуратным.

---

### 2. **Netlify**

- **Цена:** бесплатно (тар Free); платно от $19/мес.
- **Плюсы:**  
  - Простой интерфейс, деплой по Git или загрузке папки.  
  - CDN, бесплатный HTTPS, формы и функции (Netlify Functions).  
  - Хорошая скорость и надёжность.
- **Минусы:** API для Telegram нужно перенести в формат Netlify Functions (отдельная папка и сигнатура), либо использовать их Forms + webhook.
- **Итог:** отличное соотношение цена/качество; если готовы один раз адаптировать `api/send-lead` под Netlify — будет не хуже Vercel.

---

### 3. **Cloudflare Pages**

- **Цена:** бесплатно; платные тарифы для продвинутых сценариев.
- **Плюсы:**  
  - Бесплатный хостинг статики, отличный CDN.  
  - Бесплатный SSL, быстрая загрузка по всему миру.  
  - Можно подключить Workers (аналог serverless) для API заявок.
- **Минусы:** для отправки в Telegram нужна отдельная настройка Worker’а (переписать логику из `api/send-lead.js`).
- **Итог:** если хочется максимум скорости и бесплатно — Cloudflare Pages + Worker дадут «конфетку», но потребуется небольшая доработка бэкенда.

---

### 4. **GitHub Pages**

- **Цена:** бесплатно.
- **Плюсы:** просто, бесплатно, удобно, если проект уже в GitHub.
- **Минусы:** только статика. Нет serverless — формы не смогут напрямую слать заявки в Telegram. Придётся использовать внешний сервис (Formspree, Telegram Bot API через другой сервис) или редирект на форму третьей стороны.
- **Итог:** подходит, если решите обрабатывать заявки через Formspree/webhook; сам сайт будет открываться быстро и стабильно.

---

### 5. **Timeweb, Beget, Reg.ru (классический хостинг в РФ)**

- **Цена:** от ~100–200 ₽/мес за простой тариф.
- **Плюсы:**  
  - Русская поддержка, оплата в рублях.  
  - Обычно есть панель (cPanel или аналог), можно залить файлы по FTP.  
  - Свой домен часто в комплекте или со скидкой.
- **Минусы:**  
  - Нет встроенного serverless — для `api/send-lead.js` нужен либо отдельный маленький VPS/Node-скрипт, либо замена на Formspree/внешний webhook.  
  - Скорость и CDN чаще хуже, чем у Vercel/Netlify/Cloudflare.
- **Итог:** хороший вариант, если важно «всё в рублях» и привычная панель; для «конфетки» по скорости лучше дополнительно использовать CDN или рассмотреть Vercel/Netlify с российским доменом.

---

### 6. **VPS (Selectel, Timeweb Cloud, DigitalOcean и т.п.)**

- **Цена:** от ~200–400 ₽/мес за минимальный VPS.
- **Плюсы:** полный контроль: можно поднять Node.js, настроить Nginx, свой API для Telegram.
- **Минусы:** нужно самому настраивать сервер, SSL, обновления; для одного лендинга обычно избыточно.
- **Итог:** имеет смысл, если планируете несколько сайтов или сложный бэкенд; для одного лендинга проще и дешевле Vercel/Netlify.

---

## Рекомендация под ваш проект

- **Чтобы сайт был «конфеткой» по скорости и простоте:**  
  **Vercel** — у вас уже готовая конфигурация и API, достаточно задеплоить и прописать переменные Telegram.

- **Если хотите бесплатно и максимально быстро по миру:**  
  **Cloudflare Pages** + переписать отправку заявок в Cloudflare Worker.

- **Если важна оплата в рублях и привычный хостинг:**  
  **Timeweb / Beget** для статики + Formspree (или другой сервис с webhook в Telegram) для форм.

---

## Краткий чек-лист перед запуском

- [ ] Решить, где хостить (оптимально: Vercel).
- [ ] Создать Telegram-бота, получить токен и Chat ID.
- [ ] Задеплоить проект (Git или ZIP).
- [ ] Указать на хостинге `TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID`.
- [ ] Проверить отправку заявки с сайта в Telegram.
- [ ] При необходимости купить домен и привязать его в настройках проекта.

После этого сайт будет доступен в интернете, формы будут приносить заявки в Telegram, а при выборе Vercel/Netlify/Cloudflare — страницы будут грузиться очень быстро.
